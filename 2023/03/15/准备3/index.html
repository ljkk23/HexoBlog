什么是spring AOP

aop 是面向切面编程，他是对面向对象编程的补充，为什么有他呢，因为在编程过程中，会有一些不是核心业务逻辑所关心的，但是有必须有的一段代码，而且是很多业务都需要的，所以为了使程序员能够着重关注核心业务逻辑以及增加了代码复用，所以就有了aop,应用场景就是日志、鉴权等，那么他是怎么实现的呢，他将这部分共用的代码封装为一个切面，然后织入到对应需要的切入点，他是基于代理实现的，spring 支持的是jdk动态代理和cglib动态代理，jdk需要实现接口，cglib’不需要实现接口，他是基于ASM原理，他里面的组成部分，首先定义一个切点aspect,然后通过pointCut通过切入表达式去定义哪些需要切入，这里就有两种，一种是基于路径，一种是基于注解，然后有5种通知，前置、返回后，异常后、后置、坏绕，环绕可以通过jonitpoint去实现这这几种，还可以通过jonitpoint.getsignature去获取该方法的方法名等

自定义注解：target 可以注解到哪些上面，retention在什么时候织入

@Slf4j
@Aspect
@Component
public class ControllerAop {

    // 切入点表达式：用来指定哪些类需要代理
    @Pointcut("execution(* com.balloon..controller.*(..))")
    public void controllerPointcut() {
    }

    @Around("controllerPointcut()")
    public Object around(ProceedingJoinPoint joinPoint) throws Throwable {
        long start = System.currentTimeMillis();
        boolean hasException = false;
        String method = joinPoint.getSignature().toShortString();
        Result<?> result = null;
        try {
            Object obj = joinPoint.proceed();
            if (obj instanceof Result) {
                result = (Result<?>) obj;
            }
            return obj;
        } catch (Throwable e) {
            log.error("[Controller] {}执行异常, param={}", method, JSONUtil.toJSONString(joinPoint.getArgs()), e);
            hasException = true;
            throw e;
        } finally {
            String cost = String.valueOf(System.currentTimeMillis() - start);
            String suc = "-", code = "-", msg = "-";
            if (result != null) {
                suc = String.valueOf(Result.isSuccess(result));
                code = String.valueOf(result.getCode());
                msg = result.getMsg();
            }
            log.info("[Controller] {}执行结束, 耗时{}ms, exception={}, suc={}, code={}, msg={}", method, cost, hasException, suc,
                    code, msg);
        }
    }

}

类加载

加载：通过全类名获取定义此类的二进制字节流，将流中的静态存储结构转换为方法区的数据结构，将流中的常量池转换为运行时常量池，生成一个class类对象供外部访问，比如反射

验证：为了保证字节流中的信息是安全的，保证这些信息当作代码运行之后不会危害虚拟机安全，主要验证class文件格式、字节码语义以及符号引用是否正常

准备：为类的静态变量分配空间以及设置默认初始值

解析：将符号引用转换成直接引用，也就是得到类和方法、字段在内存中的指针以及偏移量

初始化：在初始化的时候，为了确保在多线程下的安全性，在初始化cinit的时候会带有初始化锁LC,只有第一次使用主动去使用一个类的时候才会去加载，有几种情况出发初始化，

  • new getStatic putStatic以及invokestatic,即遇到new 对象以及访问静态字段以及调用静态方法，
  • 用反射的时候，比如class.forName以及newInstance
  • 初始化子类的时候需要先初始化父类

类卸载：

条件：该类的所有实例对象以及没有引用和类加载器被GC

new一个对象和clone 一个对象的区别

new 一个对象是当遇到new 之后，去看new 后面的类型，然后分配内存之后，再调用构造函数，再对其进行属性填充，然后将引用发布到外部。

clone 是分配内存之后，将原对象的值复制给新对象的属性
