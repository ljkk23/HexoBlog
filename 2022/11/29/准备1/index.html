21：合并两个有序链表：

class Solution {
    public ListNode mergeTwoLists(ListNode list1, ListNode list2) {
        ListNode head=new ListNode();
        ListNode tmp=head;
        
        while(list1!=null &&  list2!=null){
            if(list1.val <= list2.val){
                tmp.next=list1;
               tmp=tmp.next;
                list1=list1.next;
            }else{
                tmp.next=list2;
               tmp=tmp.next;
                list2=list2.next;
            }
        }
        if(list1==null){
             tmp.next=list2;
        }else if(list2==null){
             tmp.next=list1;
        }
        return head.next;
    }
}
  • 不能tmp=list1，应该是tmp.next=list1,因为tmp已经被赋予了head,所以如果你在tmp=list的话。就会导致head不是结果链表的头结点。
  • 这种题注意边界处理，就是list为空，应该首先判断

86：分隔链表

这道题和上面合并是反着的，所以这里可以反着思考，遍历一次将小于x的放在一个新建的链表中，大于x的放在另一个新建的链表中。

  • 因为这里需要新建链表，所以在拷贝node节点时，要新建一个node,将head的值拷贝过去，不能直接max=head,max=max.next,因为是浅克隆，所以这里max直接进入了head链，因为还要操作小于x的，如果像这样一直在原本的head链操作，就会出现问题

  • 深克隆的实现方式：手动实现cloneable接口，然后重写clone方法

剑指offer 22 找到链表中环的入口

用快慢指针也可以做，需要首先找到快慢指针的相遇点，然后将快慢指针中一个放到head去，直到两个指针再次相遇

while (fast != null && fast.next != null) {
        fast = fast.next.next;
        slow = slow.next;
        if (fast == slow) break;
    }
    // 上面的代码类似 hasCycle 函数
    if (fast == null || fast.next == null) {
        // fast 遇到空指针说明没有环
        return null;
    }

    // 重新指向头结点
    slow = head;
    // 快慢指针同步前进，相交点就是环起点
    while (slow != fast) {
        fast = fast.next;
        slow = slow.next;
    }

92 反转链表2

1.采用迭代的方法，这里就在反转链表1的迭代方法上，找到开始反转节点的前节点leftPre和反转节点结束的右节点rightNext，然后将 pre节点设置为rightNext，最后反转之后将反转之后的头节点和leftPre接上。

2.递归：首先需要知道反转前n个，然后这道题是在这个基础上的，反转前n个，就是需要把head.next=第n+1个，所以，在递归第一步就判断n是否为一，找到后驱节点

ListNode successor = null; // 后驱节点

// 反转以 head 为起点的 n 个节点，返回新的头结点
ListNode reverseN(ListNode head, int n) {
    if (n == 1) {
        // 记录第 n + 1 个节点
        successor = head.next;
        return head;
    }
    // 以 head.next 为起点，需要反转前 n - 1 个节点
    ListNode last = reverseN(head.next, n - 1);

    head.next.next = head;
    // 让反转之后的 head 节点和后面的节点连起来
    head.next = successor;
    return last;
}

反转区间：

ListNode reverseBetween(ListNode head, int m, int n) {
    // base case
    if (m == 1) {
        return reverseN(head, n);
    }
    // 前进到反转的起点触发 base case
    head.next = reverseBetween(head.next, m - 1, n - 1);
    return head;
}
25 k个一组翻转链表

同样有迭代和递归两种，不同的是在主逻辑外专门写一个反转函数，

这里主要讲一讲递归：

在reverse方法中反转m,到n,因为返回的是反转后的头节点，所以需要连接起来

/** 反转区间 [a, b) 的元素，注意是左闭右开 */
ListNode reverse(ListNode a, ListNode b) {
    ListNode pre, cur, nxt;
    pre = null; cur = a; nxt = a;
    // while 终止的条件改一下就行了
    while (cur != b) {
        nxt = cur.next;
        cur.next = pre;
        pre = cur;
        cur = nxt;
    }
    // 返回反转后的头结点
    return pre;
}

主逻辑：

ListNode reverseKGroup(ListNode head, int k) {
    if (head == null) return null;
    // 区间 [a, b) 包含 k 个待反转元素
    ListNode a, b;
    a = b = head;
    for (int i = 0; i < k; i++) {
        // 不足 k 个，不需要反转，base case
        if (b == null) return head;
        b = b.next;
    }
    // 反转前 k 个元素
    ListNode newHead = reverse(a, b);
    // 递归反转后续链表并连接起来
    a.next = reverseKGroup(b, k);
    return newHead;
}
