第一天

准确来说，确定写这个项目以及确定架构和技术栈和功能模块才是第一天，因为花了一天的时间在网上找各大博主推荐的项目的架构功能，才确定了这个项目。

话不多说，首先建项目，因为看到使用的是idea自带的maven，所以在网上搜了下有没有什么影响，结果就是说自带的maven没有在命令行的maven命令，只能在idea集成的可视化界面进行操作，因为当前我只针对这个java项目，不涉及其他的什么自定义编译或者编译打包其他项目，所以就用这个吧，毕竟也有了之前的项目的pom的缓存，在引入时也会快一些，可能也不会快在那里去，因为我也忘记了之前项目的版本号哈哈哈哈，就这样吧。

引入父工程的pom

看到了之前项目的一个叫spring-boot-starter-actuator的依赖项，就去看了到底是什么，结果是一个提供对当前应用的监控的服务，引入他之后，可以在yml中配置暴露的端点以及权限的控制，之前在尚硅谷的cloud教程中就说到要引入这个接口以便nacos等监控（好像是这样，待会试一试），看不引入会有什么影响，

1.在父级模块的pom文件中都会有pom这一句，那么为什么会有呢：

以下配置pom的意思是使用maven分模块管理，都会有一个父级项目，pom文件一个重要的属性就是packaging（打包类型），一般来说所有的父级项目的packaging都为pom，packaging默认类型jar类型，如果不做配置，maven会将该项目打成jar包。

2.因为有一些全局的工具类和返回的response以及一些公共类，所以这里就建了一个common的包，然后这里需要在pom文件中加上对common包的依赖

3.用parent继承spring-boot-starter-paren，有什么用呢

<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.2.1.RELEASE</version>
    <relativePath/> <!-- lookup parent from repository -->
</parent>

通过继承spring-boot-starter-parent，默认具备了如下功能：

  1. Java版本（Java8）
  2. 源码的文件编码方式（UTF-8）
  3. 依赖管理
  4. 打包支持
  5. 动态识别资源
  6. 识别插件配置
  7. 识别不同的配置，如：application-dev.properties 和 application-dev.yml

以上继承来的特性有的并非直接继承自spring-boot-starter-parent，而是继承自spring-boot-starter-parent的父级spring-boot-dependencies

他与下面这个功能不同的是，parent只是指定了一些springboot的版本，如果子板块还是需要引入springboot-starter

<!-- SpringBoot的依赖配置-->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-dependencies</artifactId>
    <version>2.5.0</version>
    <type>pom</type>
    <scope>import</scope>
</dependency>

搭建骨架

1.在common包下新建Response的返回json：

@JsonInclude(JsonInclude.Include.NON_NULL)

这 一行代码是jackson序列化的机制，将属性为null的属性不序列化。

Jackson 的核心模块由三部分组成：

  • jackson-core，核心包，提供基于“流模式”解析的相关 API，包括 JsonPaser 和 JsonGenerator。
  • jackson-annotations，注解包，提供标准的注解功能；
  • jackson-databind ，数据绑定包，提供基于“对象绑定”解析的相关 API （ ObjectMapper ） 和基于“树模型”解析的相关 API （JsonNode）。

要想使用 Jackson，需要在 pom.xml 文件中添加 Jackson 的依赖。

<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>2.10.1</version>
</dependency>

jackson-databind 依赖于 jackson-core 和 jackson-annotations，所以添加完 jackson-databind 之后，Maven 会自动将 jackson-core 和 jackson-annotations 引入到项目当中。

2.mapper的接口和映射文件的位置：

1.不做多余配置的前提下,一定要保证mapper接口和xml映射文件的相对路径一致;

2.如果xml映射文件存放在resources下,而且mapper接口和xml映射文件的相对路径不一致,需要在application.yml中进行mapper-locations配置

yml文件中添加

mybatis-plus:
  mapper-locations: classpath:mappers/*.xml

3.如果xml映射文件存放在java下,不仅要在application.yml中进行mapper-locations配置,同时还要在pom.xml文件中添加相关配置

<!-- 项目打包时会将java目录中的*.xml文件也进行打包 -->
<build>
    <resources>
        <resource>
            <directory>src/main/java</directory>
            <includes>
                <include>**/*.xml</include>
            </includes>
            <filtering>false</filtering>
        </resource>
    </resources>
</build>
