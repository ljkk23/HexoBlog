昨天才遇到的mapper的位置问题

昨天才遇到的，今天又忘记了，使用的是pom文件配置加classpath,但是还是错了不知道为什么，那就简单一点吧，就添加mapper注解吧。

INSERT_UPDATE设置自动填充，需要在填充器里面加

@Override
   public void insertFill(MetaObject metaObject) {

       this.setFieldValByName("createTime", LocalDateTime.now(), metaObject);
       this.setFieldValByName("updateTime", LocalDateTime.now(), metaObject);

   }

   @Override
   public void updateFill(MetaObject metaObject) {
       this.setFieldValByName("updateTime", LocalDateTime.now(), metaObject);
   }

insertfill是指在insert时会填充的字段，updateFill是指在update时改变的字段

feign、openfeign

在application上加上注解@EnableDiscoveryClient,使其注册服务进nacos,在需要调用其他服务的application上加上注解EnableFeignClients

@EnableDiscoveryClient
//开启远程调用openfeign
@EnableFeignClients

这里遇到一个错误：在通过openfeign调用远程的client的时候，报了405的错误，

被调用接口是个Get请求，参数没加@RequestParam注解，调用接口的参数也没有加@RequestParam注解，导致报错。

@GetMapping("/base/user/getUserList")
List<User> getUserList(User user);

解决办法：参数加@RequestParam注解即可。

@GetMapping("/base/user/getUserList")
List<User> getUserList(@RequestParam User user);

系统认识远程调用

参考：https://blog.csdn.net/a745233700/article/details/122916856#t9

最开始是restTemplate加loadbalence负载均衡发起http请求，后来是riibon集成了restTeplate加loadBalence，只在restTemplate上加一个注解就可以实现多种负载均衡，方便。后来就是feign

Feign 就是一个 Http 客户端的模板，目标是减少 HTTP API 的复杂性，希望能将 HTTP 远程服务调用做到像 RPC 一样易用。Feign 集成 RestTemplate、Ribbon 实现了客户端的负载均衡的 Http 调用，并对原调用方式进行了封装，使得开发者不必手动使用 RestTemplate 调用服务，而是声明一个接口，并在这个接口中标注一个注解即可完成服务调用，这样更加符合面向接口编程的宗旨，客户端在调用服务端时也不需要再关注请求的方式、地址以及是 forObject 还是 forEntity，结构更加明了，耦合也更低，简化了开发
因为feign停止维护了，所以就有了openFeign

openFeign 作为微服务架构下服务间调用的解决方案，是一种声明式、模板化的 HTTP 的模板，使 HTTP 请求就像调用本地方法一样，通过 openFeign 可以替代基于 RestTemplate 的远程服务调用，并且默认集成了 Ribbon 进行负载均衡。

public interface OpenFeignService {
    /**
     * 必须要@RequestParam注解标注，且value属性必须填上参数名
     * 方法参数名可以任意，但是@RequestParam注解中的value属性必须和provider中的参数名相同
     */

    String test(@RequestParam("id") String arg1,@RequestParam("name") String arg2);
}
