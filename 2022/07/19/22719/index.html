  • 当我在使用七牛云实现对象存储OSS时，需要在application.yaml中实现oss的认证信息时，（为什么需要在yaml中配置，前面也说了是java反射的机制），因为这是自己从外面定义的，所以不会自动注入容器，（如果是spring.datasource 的配置，spring就会自动注入容器，方便mp找到），所以这里需要在application中配置两个注解：

    @ConfigurationPropertiesScan
@EnableConfigurationProperties

    对应的yaml：

    oss:
  accessKey: ieL1Biy32ALYzqRbUu-4FF36nnvwO3-WTXiMsyTf
  secretKey: kcImZkWbcK4uiWUtwT1BJ2RMN9-M5TItBXtqQOsx
  bucket: lj-blog

    同时需要在oss配置中，加上@ConfigurationProperties(prefix = “oss”)，他会自动读取accessKey这几个参数，并赋值进去

    @Service
@Data
@ConfigurationProperties(prefix = "oss")
public class UploadServiceImpl implements UploadService {
    private String accessKey;
    private String secretKey;
    private String bucket;
    
    private String upload(MultipartFile file, String path) {
        //构造一个带指定 Region 对象的配置类
        Configuration cfg = new Configuration(Region.autoRegion());
        //...其他参数参考类注释

        UploadManager uploadManager = new UploadManager(cfg);
        //...生成上传凭证，然后准备上传
//        String accessKey = "your access key";
//        String secretKey = "your secret key";
//        String bucket = "sg-blog";

        //默认不指定key的情况下，以文件内容的hash值作为文件名
        String key = path;

        try {
//            byte[] uploadBytes = "hello qiniu cloud".getBytes("utf-8");
//            ByteArrayInputStream byteInputStream=new ByteArrayInputStream(uploadBytes);


            InputStream inputStream = file.getInputStream();
            Auth auth = Auth.create(accessKey, secretKey);
            String upToken = auth.uploadToken(bucket);

            try {
                Response response = uploadManager.put(inputStream,key,upToken,null, null);
                //解析上传成功的结果
                DefaultPutRet putRet = new Gson().fromJson(response.bodyString(), DefaultPutRet.class);
                System.out.println(putRet.key);
                System.out.println(putRet.hash);
                return "http://rcocs0mr3.hn-bkt.clouddn.com/"+key;
            } catch (QiniuException ex) {
                Response r = ex.response;
                System.err.println(r.toString());
                try {
                    System.err.println(r.bodyString());
                } catch (QiniuException ex2) {
                    //ignore
                }
            }
        } catch (Exception ex) {
            //ignore
        }
        return "www";
    }

    @Override
    public ResponseResult uploadImg(MultipartFile img) {
        //判断文件类型是否为png
        String originalFilename = img.getOriginalFilename();
        if (!originalFilename.endsWith(".png")){
            throw new SystemException(AppHttpCodeEnum.FILE_TYPE_ERROR);
        }
        //TODO 判断文件大小
        //上传文件
        //生成保存路径
        String path= PathUtils.generateFilePath(originalFilename);
        String ImgUrl = upload(img, path);
        return ResponseResult.okResult(ImgUrl);

    }

}

  • ssh @通过ssh连接服务器，连接服务器的话，自动就安装了openssh，所以就只需要在服务器的/etc/ssh/sshd_config文件设置PermitRootLogin yes 以及PubkeyAuthentication yes以及RSAAuthentication yes，然后用ssh root@150.158.141.30登陆，如果需要用密钥就需用在本机ssh-keygen -t rsa生成密钥对，然后将本机的id_rsa.pub这个为公钥将其放在服务器用户的ssh下，如果是root，就在/root/.ssh下，如果没有目录就创建，然后放在authorkey里面，然后记住需要重启ssh服务，sudo /etc/init.d/ssh restart，注意还有可能需要对文件改权限，chmod 600 authorzed_keys以及chmod 700 /root/.ssh，然后再用本机连接

    文章参考：https://www.runoob.com/w3cnote/set-ssh-login-key.html
